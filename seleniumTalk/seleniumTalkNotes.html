    • Pytest

    1. Problems with unittest:
      • lot of boiler plate.
      • Not very pythonic (Clone of Java's JUnit etc)
      • Has a mess of Assert... methods
    2. The Pytest approach
      • Utilize Python's assert statement instead of assert...
      • Not as class centric. Don't need to stick everything in a TestCase class.
      • fixture functions instead of setUp and tearDown methods
    3. pip install pytest
    4. Create a subdirectory for tests within your virtual environment. If you don't do this, pytest will try to run any tests it finds in the packages you've installed into your virtual environment.
    5. pytest will look for files named either *_test.py or test_*.py
    6. Open test_hello.py at this point
    7. from those files pytest will run test_ functions either free-floating test_* functions or test_ methods in Test... classes which don't have an __init__ method.
    8. run pytest --help
    9. run pytest --collect-only
    10. run pytest test_hello.py
    11. Options when testing web apps
      • baseUrl - So you can switch between environments (i.e. development vs. testing)
      • what browser the tests run with
    12. Open conftest.py (pytest looks for this file).
    13. run pytest --help Point out custom options
    14. Open test_webapp.py
    15. Fixtures
      1. arguments to test_ function are fixtures
      2. pytest framework looks for fixture with matching name.
      3. fixtures (indicated by @fixture decorator) take place of setup and teardown methods
      4. Scope can be function, class, module or session
      5. Pytest will execute tests which use a particular together
      6. point out generator behavior of driver fixture
      7. point out how options are being consumed in the two fixtures
      8. Fixtures can call other fixtures see customerID fixture in fixtureSnippet
